# 部署指南（一次性，约 15 分钟，全程网页操作，不需要命令行）

完成后你得到：一个公网网址的网站，每天早上 06:30（北京时间）自动更新数据、
自动核实入库，周日自动出周报、每月 1 日出月报、季度和年度自动出对应报告。

## 前提

一个 GitHub 账号（github.com 注册，免费）。国内访问 GitHub 时好时坏，
部署这一次需要能打开 github.com；日常看网站访问的是 `*.github.io`。

## 第一步：建仓库

1. 登录 GitHub → 右上角 ＋ → **New repository**
2. Repository name 填 `musk-intel-dashboard`（或任意名字）
3. 选 **Public**（免费 Pages 的要求；隐私影响见 README 末尾）
4. 其他都不勾，点 **Create repository**

## 第二步：上传文件

1. 在新仓库页面点 **uploading an existing file**（或 Add file → Upload files）
2. 把 `站点文件夹` 文件夹里的**全部内容**（不是文件夹本身）拖进去：
   `index.html`、`data.js`、`README.md`、`部署指南.md`、`.nojekyll`、
   `scripts` 文件夹、`.github` 文件夹
   - 注意：网页拖拽上传时，直接把这些文件和文件夹一起框选拖入即可，目录结构会保留
   - 若 `.github`、`.nojekyll` 这类点开头的文件在你的资源管理器里看不到，
     先在"查看"里勾选"隐藏的项目"
3. 底部点 **Commit changes**

## 第三步：开启 Actions 写权限

1. 仓库页 → **Settings** → 左栏 **Actions** → **General**
2. 拉到底部 **Workflow permissions** → 选 **Read and write permissions** → Save
   （否则机器人无法把抓到的数据提交回仓库）

## 第四步：开启 Pages

1. **Settings** → 左栏 **Pages**
2. Source 选 **Deploy from a branch**；Branch 选 **main**，目录 **/(root)** → Save
3. 一两分钟后本页会显示你的网址：`https://<你的用户名>.github.io/musk-intel-dashboard/`

## 第五步：首次运行验证（三分钟自检）

1. 仓库页 → **Actions** 标签 → 左栏点"自动更新数据与报告" → **Run workflow**
   - 想立即看到周报效果，可在 force_report 里填 `weekly` 再 Run
2. 等待绿色对勾（约 1 分钟）。红叉则点进去看日志，把报错内容发给 AI 助手
3. 打开你的网址：
   - 右栏【数据健康】四个源应为绿灯
   - 【决策档案】页应出现"自动核实审计"记录
   - 填了 force_report 的话，【报告】页应有一份周报

全部通过 = 部署完成。之后不需要任何操作，每天 06:30 自动跑。

## 日常使用

| 频率 | 动作 |
|---|---|
| 随时 | 打开网址看站（顶栏显示数据更新时间） |
| 每周 | 【报告】页看自动生成的周报 |
| 不定期 | 【决策档案】页复核自动入库审计，发现误判用"[冲正]"条目修正 |
| 每周日 | 【决策档案】页点【导出备份】（批注和决策日志存在浏览器里，这步不能省） |

## 出事时

| 现象 | 处置 |
|---|---|
| 网站数据多天不更新 | 仓库 Actions 页看最近运行是否红叉；红叉日志发给 AI 助手。另：GitHub 会自动暂停 60 天无人访问仓库的定时任务，进 Actions 页点一下 Enable 即恢复 |
| 某数据源持续红灯/熔断 | 周报头条会提示；把 诊断日志.txt 内容与 scripts/update.py 发给 AI 助手修复，改完在网页端直接编辑替换该文件即可 |
| 数据损坏/网站白屏 | 仓库页 → data.js → History，找最近正常版本 → Revert（git 历史就是备份，无限份） |
| 批注/决策日志丢失 | 【决策档案】→【导入备份】，选你上次导出的 JSON |

- 改聚簇灵敏度：`SIM_THRESHOLD`（调低 → R2 更易触发，但误合并风险升高）
